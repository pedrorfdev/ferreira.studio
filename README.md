## ✦ Sobre

O `ferreira.studio` não foi pensado como um portfólio tradicional de desenvolvedor.

O projeto foi construído como uma experiência imersiva e orientada por estado, onde:

* interface
* arquitetura
* motion
* narrativa
* e visão de produto

funcionam como partes do mesmo sistema.

O objetivo não é apenas exibir projetos, mas comunicar:

* profundidade técnica
* visão de produto
* sensibilidade em UX
* arquitetura frontend
* cuidado com engenharia

## ✦ Arquitetura da Experiência

### App Shell Architecture

A aplicação mantém a interface principal montada durante toda a navegação.

Ao invés de carregar múltiplas páginas desconectadas, a experiência funciona como um produto contínuo:

* layouts persistentes
* overlays animados
* transições fluidas
* contexto visual compartilhado

---

### Navegação orientada por estado

A navegação é controlada por estado ao invés de carregamentos tradicionais de página.

Isso cria:

* transições cinematográficas
* UX mais fluida
* continuidade visual
* interações imersivas

---

### Motion System

As animações seguem uma abordagem minimalista e editorial.

Motion é utilizado para:

* hierarquia
* ritmo
* feedback
* narrativa

E não apenas decoração.
